## 目标

- 把“写单题题解”做成可执行脚本。
- 脚本使用 `.env` 中的模型配置和 API 凭据请求大模型。
- 保证单题只发起一次主请求；若样例编译或运行失败，再附上样例检测结果重新请求该题。
- 只有样例全部通过后，才视为该题完成并落盘。

## 约束与决策

- 优先复用现有题面目录 `atcoder-output/<contest>/en/*.en.md` 作为输入，而不是额外联网取题面。
- 题解结构与 `C++` 代码风格继续遵守当前全局 `algorithm-editorial-reference`。
- 验证链路需要覆盖：
  - 从题面中提取样例输入 / 输出
  - 从模型返回的 Markdown 中提取 `cpp` 代码块
  - 编译并运行
  - 输出逐样例对比结果
- 失败重试时，要把编译错误或样例对比结果回灌给模型。
