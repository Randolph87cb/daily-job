## 2026-05-07 补充更新 6

- 新发现问题：
  - 虽然 OpenAI 翻译已可正常完成，但双语输出仍经常退回为“整篇中文 + 整篇英文”的 fallback 形式，而不是逐段穿插对照。
- 根因：
  - 旧实现依赖“整篇 Markdown 翻译后，块数与原文完全一致”。
  - 模型有时会轻微合并段落、调整空行或改写标题，导致块数不一致，于是触发 fallback。
- 修复思路：
  - 保持“一题一次 API 请求”不变。
  - 但不再让模型直接翻整篇 Markdown。
  - 改为本地先按块拆分，再把所有需要模型翻译的块打包成一份 JSON，一次请求交给模型返回逐块翻译结果。
  - 本地再按块 id 回填，这样逐段对照不会因为模型改空行而失配。
- 修改文件：
  - `D:\workspace\日常事务\atcoder-arc218\scripts\translate_markdown.py`
- 关键实现：
  - 新增 `should_translate_with_model()`：标题、来源、分值、代码块不送模型。
  - 新增 `translate_block_locally()`：本地处理无需模型的块。
  - `OpenAITranslator` 新增逐块 JSON 翻译模式：
    - 单次请求发送 `{"blocks":[{"id":1,"text":"..."},...]}` 
    - 要求模型原样返回同结构 JSON
    - 本地按 `id` 回填并恢复公式 / URL / inline code 占位符
  - `translate_markdown()` 在 OpenAI 模式下改为：
    - 先拆块
    - 再单次请求批量翻译需要翻译的块
    - 最后重组为完整译文
