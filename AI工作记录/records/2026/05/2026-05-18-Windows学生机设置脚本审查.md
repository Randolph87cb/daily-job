## 审查发现

- `Set-StudentSafeDefaults.ps1` 使用 `Ensure-RegistryKey` 创建 `backups` 目录，但该函数内部调用的是 `New-Item -Path $Path -Force`，在文件系统提供器下会默认创建普通文件而不是目录；首次运行时，`backups` 很可能变成文件，后续写入 `backups/changes-时间戳.json` 会失败。
- `Run-Set-StudentSafeDefaults.cmd` 当前无条件追加 `-ApplyToDefaultUser -RestartExplorer`，意味着双击包装器时总会请求管理员权限并修改默认用户配置文件，行为范围比 README 中“尤其是在需要 `-ApplyToDefaultUser` 时”的表述更大。
- 脚本声明了 `SupportsShouldProcess`，但 `Set-RegistryDword` / `Set-RegistryBinary` 在 `ShouldProcess()` 返回 `false` 时仍然会写日志并输出 `Set:`，导致 `-WhatIf` 模式下日志与实际执行不一致。
- 项目根目录新增了长期保留的 `windows-student-guard/`，但根目录 `README.md` 的目录结构与目录说明尚未同步；同时 `.gitignore` 里也还没有忽略 `windows-student-guard/backups/` 这类运行产物。

## 验证与依据

- 语法检查：
  - 使用 PowerShell Parser 解析 `Set-StudentSafeDefaults.ps1`，结果 `PARSE_OK`。
- 最小实验：
  - 在临时目录执行 `New-Item -Path <missing-path> -Force`，返回类型为 `System.IO.FileInfo`，确认当前实现会创建文件而不是目录。
- Git 状态：
  - 当前仅看到未跟踪目录 `windows-student-guard/`。

## 后续建议

1. 把 `backups` 创建逻辑改为显式目录创建，例如 `New-Item -ItemType Directory`。
2. 把包装器拆成“当前用户版”和“含默认用户版”，或至少让 README 与实际行为保持一致。
3. 如果保留 `SupportsShouldProcess`，要让 `-WhatIf` 下的日志、返回值和真实执行一致。
4. 若该目录准备长期保留，补根目录 `README.md` / `AGENTS.md` / `.gitignore`。

## 本次实现

- 已把 `Set-StudentSafeDefaults.ps1` 重构为：
  - 区分注册表路径创建与文件目录创建，修复 `backups/` 首次运行可能被建成文件的问题。
  - 为注册表写入补充 `String` 类型设置函数，并修正 `ShouldProcess` 逻辑，避免 `-WhatIf` 下出现伪变更日志。
  - 新增当前用户鼠标速度设置，默认恢复为 `Windows` 中间档 `10`。
  - 新增鼠标指针方案设置，默认使用 `Windows Black (large)`，提升在教室环境下的可见性。
  - 新增主显示器分辨率调整逻辑，优先读取 `root\wmi` 下的显示器推荐模式，拿不到时再回退到最高支持模式，并通过 `ChangeDisplaySettings` 应用。
  - 继续支持 `-ApplyToDefaultUser`，将任务栏、触摸板、鼠标速度和鼠标指针方案写入默认用户配置文件。
- 已同步文档：
  - `windows-student-guard/README.md`
  - 根目录 `README.md`
  - 根目录 `AGENTS.md`
  - 根目录 `.gitignore`

## 实现决策

- 分辨率没有写死成 `1920x1080` 这类固定值，而是优先选择“当前主显示器推荐模式”。
  - 原因是 LCD/LED 屏幕在低于原生分辨率时通常会发糊。
  - 如果后续发现个别高 DPI 设备需要更大界面，更合理的补充方向是显示缩放，而不是继续降低分辨率。
- 鼠标指针没有直接改“自定义颜色”注册表，而是复用系统内置方案 `Windows Black (large)`，减少依赖当前机器已有生成文件的风险。
