# ABC457 F - Second Gap 题解

## 题目信息

- 原题链接：[F - Second Gap](https://atcoder.jp/contests/abc457/tasks/abc457_f)
- 题面对照：[../zh-CN/abc457_f.zh-CN.md](../zh-CN/abc457_f.zh-CN.md)
- 分值：525

## 题意概括

给定长度为 $N - 1$ 的数组 $D$，要求统计有多少个排列 $P$ 满足：

- 对每个后缀 $(P_i, P_{i+1}, \ldots, P_N)$，
- 其中最大值和第二大值所在位置的距离恰好是 $D_i$。

答案对 $998244353$ 取模。

## 解题思路

### 1. 从后往前做“插入 DP”

考虑从后缀开始构造排列。

假设我们已经处理好了后缀 $(P_{i+1}, \ldots, P_N)$，现在要在最前面插入一个新元素 $P_i$。  
这里只关心相对大小，不关心具体数字，这和经典的插入 DP 是一样的想法。

对于当前后缀，最重要的是：

- 最大值在哪个位置；
- 第二大值在哪个位置。

如果把这两个位置记成 $\{a, b\}$，那么插入 $P_i$ 后有三种情况：

1. $P_i$ 成为新的最大值  
   此时新的前两大位置变成 $\{i, a\}$。

2. $P_i$ 成为新的第二大值  
   此时新的前两大位置变成 $\{a, i\}$。

3. $P_i$ 既不是最大值，也不是第二大值  
   此时新的前两大位置仍然是 $\{a, b\}$。

### 2. 为什么只记录“最大值的位置”就够了

朴素想法会想记录 $(a, b)$ 两个位置，但其实没必要。

原因是：

- 在前两种转移里，只要知道原来的最大值位置 $a$，就已经能确定是否合法；
- 在第三种转移里，前两大位置根本不变，因此要想继续满足条件，只要求 $D_i = D_{i+1}$。

也就是说，第三种转移完全不依赖第二大值的位置 $b$ 的具体数值。

因此可以把状态压缩成：

- $dp[i][a]$：表示后缀 $(P_i, \ldots, P_N)$ 中，最大值出现在位置 $a$ 的方案数。

### 3. 转移怎么写

为了实现方便，下面用一维 `dp[a]` 表示当前层的状态。

初始化看长度为 $2$ 的后缀，也就是 $(P_{N-1}, P_N)$：

- 最大值可能在 $N-1$
- 也可能在 $N$

所以：

- `dp[N - 1] = 1`
- `dp[N] = 1`

接下来从 $i = N - 2$ 倒着做到 $1$。

设：

- $j = i + D_i$

因为无论 $P_i$ 成为最大值还是第二大值，原来后缀中的最大值位置都必须与 $i$ 的距离等于 $D_i$，所以那个位置只能是 $j$。

于是：

1. 若 $P_i$ 成为最大值  
   新的最大值位置变成 $i$，因此：

   - 从原来最大值在 $j$ 的状态，转移到新的 `dp[i]`

2. 若 $P_i$ 成为第二大值  
   新的最大值位置仍然是 $j$，因此：

   - 从原来最大值在 $j$ 的状态，转移到新的 `dp[j]`

3. 若 $P_i$ 既不是最大值也不是第二大值  
   只有当 $D_i = D_{i+1}$ 时才可能，因为这时“前两大位置的距离”不变。  
   此外，这种插入方式有：

   - $N - i - 1$

   种选择，因为当前后缀长度为 $N - i$，除去排在前两位的可能外，还可以插入到其余相对等级中。

### 4. 如何把总复杂度优化到 $O(N)$

如果第三种转移每次都对所有状态乘上同一个常数，那么朴素写法会是 $O(N^2)$。

但注意到：

- 第三种转移只是“把所有状态整体乘一个相同的系数”

所以可以把这个系数单独记在外面，设为 `coef`：

- `dp[a]` 只存“去掉公共系数后的值”
- 真正的状态值等于 `dp[a] * coef`

这样：

- 第三种转移只需要更新 `coef`
- 第一、二种转移只对两个位置做单点修改

最终总复杂度就能做到 $O(N)$。

## 正确性说明

### 状态定义正确

我们定义 `dp[a]` 表示当前处理到某个后缀时，“最大值出现在位置 $a$”的方案数。  
由于题目在每一层只关心最大值和第二大值之间的距离，而第二大值的位置在前两种转移中不需要单独知道，在第三种转移中又只会保持不变，因此只记录最大值位置就足够了。

### 转移一、二正确

当插入的新元素 $P_i$ 成为最大值时，原来的最大值位置必须距离 $i$ 恰好为 $D_i$，所以原来的最大值只能在 $j = i + D_i$。此时新状态的最大值位置变成 $i$，因此可以从旧状态 `dp[j]` 转移到新状态 `dp[i]`。

当插入的新元素 $P_i$ 成为第二大值时，原来的最大值位置同样必须是 $j = i + D_i$，并且新状态的最大值位置仍然是 $j$，因此可以从旧状态 `dp[j]` 转移到新状态 `dp[j]`。

### 转移三正确

当 $P_i$ 既不是最大值也不是第二大值时，后缀中的最大值位置和第二大值位置都不发生变化，所以它们之间的距离也不变。因此这种转移可行的充要条件是：

- $D_i = D_{i+1}$

一旦这个条件满足，当前元素可以放在除前两名之外的任意相对名次，总共有 $N - i - 1$ 种可能，因此所有状态都统一乘上这个系数。

### 懒乘系数正确

第三种转移对所有状态都乘同一个值，因此把它抽成公共因子 `coef` 不会改变任何状态之间的相对关系，也不会影响第一、二种转移的正确性。最终再把所有状态和乘回 `coef`，就能得到真实答案。

综上，DP 的状态、转移和优化都正确，因此算法正确。

## 复杂度

- 时间复杂度：$O(N)$
- 空间复杂度：$O(N)$

## 参考实现（C++，遵守代码规范）

```cpp
#include<bits/stdc++.h>
using namespace std;

const long long MOD = 998244353;

// 快速幂：计算 a^b mod MOD
long long qpow(long long a, long long b){

    long long res = 1;
    while(b > 0){
        if(b & 1){
            res = res * a % MOD;
        }
        a = a * a % MOD;
        b >>= 1;
    }

    return res;
}

int main(){

    ios::sync_with_stdio(false);
    cin.tie(nullptr);

    // 读取排列长度
    int n;
    cin >> n;

    // d[i] 表示第 i 层要求的距离
    vector<int> d(n + 1, 0);
    for(int i=1; i<=n - 1; i++){
        cin >> d[i];
    }

    // dp[pos] 表示去掉公共系数后，当前最大值位置为 pos 的方案数
    unordered_map<int, long long> dp;
    dp[n - 1] = 1;
    dp[n] = 1;

    // coef 记录所有状态共同乘上的系数
    long long coef = 1;

    for(int i=n - 2; i>=1; i--){
        int j = i + d[i];

        long long v = 0;
        if(dp.count(j)){
            v = dp[j];
        }

        if(d[i] == d[i + 1]){
            // 第三种转移：所有状态统一乘上 ways
            long long ways = n - i - 1;
            coef = coef * ways % MOD;

            // 由于 coef 变了，要把单点加法换算到新的系数下
            long long add = v * qpow(ways, MOD - 2) % MOD;
            dp[j] = (dp[j] + add) % MOD;
            dp[i] = (dp[i] + add) % MOD;
        } else {
            // 第三种转移不存在，旧状态全部失效，只保留前两种转移
            unordered_map<int, long long> ndp;
            ndp[j] = (ndp[j] + v) % MOD;
            ndp[i] = (ndp[i] + v) % MOD;
            dp.swap(ndp);
        }
    }

    // 把去掉的公共系数乘回去
    long long sum = 0;
    for(auto [pos, val] : dp){
        sum = (sum + val) % MOD;
    }

    long long ans = sum * coef % MOD;
    cout << ans;

    return 0;
}
```
