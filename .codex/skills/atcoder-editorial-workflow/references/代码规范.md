# 代码规范

## 变量的命名

如果题目中有要求，则按照题目要求进行命名。

- 如果题面已经给出变量名称，例如 `a1,a2,...`、`h1,h2,...`、`prices_i`、`counts_i`，参考代码应优先沿用题面命名思路。
- 当题面给出一组同类变量时，优先用数组表达，例如 `a[i]`、`h[i]`、`prices[i]`、`counts[i]`，不要改成难以对应题面的临时变量名。
- 不要为了省事把有明确含义的数据统一读进无意义变量，例如 `x`、`tmp`；除非它本身就是题面中的变量名。

- 常见变量命名：
  - 表示计数：`cnt`
  - 表示答案：`ans`
  - 表示结果：`res`
  - 累加器（求总和）：`sum`
  - 数组：`a[数组长度+5]`
  - 字符串：`string s;`
  - 字符、字符数组：`c`、`c[数组长度 + 5]`
  - 最大值：`mx`
  - 最小值：`mn`

## 示例代码默认目标

- 示例代码不仅要能通过评测，还要适合作为课堂讲解参考代码。
- 生成示例时，教学清晰度与评测正确性同等重要。
- 如果“更短的写法”和“更好讲解的写法”冲突，优先选择更好讲解的写法。

## 题解参考实现补充

- 题解里的参考代码，默认同时服务于：
  - 课堂讲解；
  - Markdown 题解展示；
  - 后续单题 PDF 与合并 PDF 导出。
- 因此参考实现应尽量保持：
  - 结构直线；
  - 注释分段清楚；
  - 变量与数组含义容易在正文中逐段对应。
- 如果确实需要保留 STL 容器，优先只保留算法真正必需的最少容器，不要在同一份教学代码里混入多种不必要容器。
- 如果把数组、辅助函数、状态表放在 `main` 外面，相关含义要在定义附近或第一次使用前给出简短注释，避免学生只看到“全局开一堆数组”却不知道用途。

## 知识讲解

### 顺序结构

- 数据类型：`int`、`double`、`long long`
- 保留小数输出：循环章节之前，使用 `cout` 进行输出，需要在 HTOJ 中给到学生提示 `cout << fixed << setprecision(n)`。在数组阶段可以使用 `printf` 进行输出。

### 选择结构

- 代码格式问题，统一按照下方给到的格式进行编程。
- 最大值、最小值数据范围：
  - 最大值：`$$mx = -1e9$$`
  - 最小值：`$$mn = 1e9$$`

### 循环结构

- 循环起始值从 `$$1$$` 开始，到 `$$n$$` 结束。
- 循环变量的选择：`i`、`j`、`k`、`l`
- 累加器 `sum` 统一赋初值为 `0` 或 `1`（除通过 `cin` 读入的变量外，其他变量统一赋初值）。

### 数组

- `char`、`string` 统一在数组章节进行讲解。
- 定义数组时，数组长度：数据范围 `+ 5`
- 能用数组表达的数据，优先不要写成 `vector`。
- 如果题目数据范围允许，数组定义优先放在 `main` 函数外面，写成全局数组，减少主函数内的干扰信息。
- 只有在数组长度无法提前稳定确定、或确实不适合用普通数组表达时，才使用 `vector`。
- 如果使用 `vector`，题解正文里要额外说明为什么这里不适合改成普通数组。
- 如果题目输入本身是两组或多组对应数据，且适合数组章节讲解，应分别存入对应数组，再进行处理。
- 不要把本应成组保存的数据，改写成“一个数组 + 一个临时变量边读边算”的形式，除非题目重点就是边读边处理。
- 在教学示例中，数组的设计要让学生能直接看出“第 i 个数据”和“第 i 个数据”的对应关系。

## 代码书写规范

### 代码格式化规范

- 统一使用以下程序框架：

```cpp
#include<bits/stdc++.h>
using namespace std;

int main(){

    return 0;
}
```

- `int main(){`、`if(条件){`、`for(...){`、`while(...){` 统一使用“左花括号不换行”的写法。
- 缩进统一使用 `4` 个空格，不使用 `Tab`。
- 运算符两侧统一保留一个空格，例如 `int ans = a + b;`、`if(x == y){`。
- `for` 循环统一写成 `for(int i=1; i<=n; i++){` 这种格式：
  - 分号后保留一个空格
  - 比较、赋值两侧不额外插入无意义空格
- 逗号后保留一个空格，例如 `int n, m;`。
- 关键逻辑块之间空一行，例如：
  - 输入部分和处理部分之间
  - 处理部分和输出部分之间
- 一行只写一条语句，不把多条语句写在同一行。
- `if`、`else if`、`else`、循环体即使只有一条语句，也统一写花括号，不省略。

### 教学注释规范

- 示例代码必须包含关键注释，不能只给纯代码。
- 注释重点包括：
  - 变量或数组的含义；
  - 输入数据存到哪里；
  - 核心处理步骤；
  - 最终输出的含义。
- 注释要简短直接，服务课堂讲解，不写重复代码表面的废话。
- 每一段关键逻辑前，至少写一条说明性注释。
- 如果题目本身有明显分步骤，注释顺序尽量对应“输入 -> 处理 -> 输出”。

### 教学可读性优先

- 同一道题如果有多种正确写法，优先选择学生更容易理解、老师更容易讲解的写法。
- 优先选择：
  - 状态变量更少的写法；
  - 处理流程更直线的写法；
  - 能用普通数组讲清楚，就不要引入 `vector`；
  - 能用全局数组讲清楚，就不要把大块数组都塞进 `main` 里；
  - 更贴近当前章节知识点的写法；
  - 更容易和样例手推对应的写法。
- 尽量避免为了代码技巧而引入额外状态判断、复杂字符串处理、过于隐含的写法。
- 尽量避免在学生基础还不稳定的示例里引入 `vector`、`set`、`map`、`unordered_map` 等容器；只有题目规模或算法结构确实需要时才使用。
- 如果必须保留容器，优先保留单个最关键容器，并让其余数据继续用普通数组表达。
- 输出格式处理时，优先使用简单稳定的方式，减少额外布尔标记控制。

### 顺序结构

#### 标准程序框架

```cpp
#include<bits/stdc++.h> // 统一用万能头
using namespace std;
int main(){

    return 0; // 程序结束
}
```

#### 保留小数输出

```cpp
// 保留两位小数输出
cout << fixed << setprecision(2) << ans;

printf("%.2lf", ans); // 统一用lf
```

#### 编程逻辑

```cpp
#include<bits/stdc++.h>
using namespace std;
int main(){
    
    // 1. 定义变量
    int a, b;
    // 2. 通过输入指令存储数据
    cin >> a >> b;
    // 3. 计算过程 (如果无此步骤可以省略)
    int ans = a + b;
    // 4. 输出结果
    cout << ans;
    
    return 0;
}
```

### 选择结构

#### 双分支

```cpp
if(条件) {
    执行语句1;
    ...
} else {
    执行语句2;
    ...
}
```

#### 多分支

```cpp
if(条件) {
    执行语句1;
    ...
} else if(条件) {
    执行语句2;
    ...
} else {
    执行语句3;
    ...
}
```

#### 最大值 && 最小值

```cpp
int mx = -1e9; // 变量名为mx，赋初值：-1e9
int mn = 1e9; // 变量名为mn，赋初值：1e9
```

### 循环结构

```cpp
for(int i=1; i<=n; i++){ // 第一层用 i
    for(int j=1; j<=n; j++){ // 第二层用 j
        for(int k=1; k<=n; k++){ // 第三层用 k
            for(int l=1; l<=n; l++){ // 第四层用 l
                执行语句1;
                ....
            }
        }
    }
}
```

### 数组

```cpp
int a[15] = {0}; // 能放全局时，优先放在 main 外面

int main(){

    int n;
    cin >> n;

    for(int i=1; i<=n; i++){
        cin >> a[i]; // 从下标为 1 的位置开始存入数据。
    }

    return 0;
}
```

```cpp
int a[15] = {0};
for(int i=1; i<=n; i++){
    cin >> a[i]; // 从下标为 1 的位置开始存入数据。
}

string s;
cin >> s; // 从下标为 0 的位置开始存入数据。

char c[15];
// ① 从下标为 0 的位置开始存入数据。
cin >> c;

// ② 逐个字符读入。
for(int i=1; i<=n; i++) {
    cin >> c[i];
}
```
